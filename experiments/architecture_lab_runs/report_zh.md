## 决策摘要

| 阶段 | 4090 多 seed 观察 | 门槛与决定 |
| --- | --- | --- |
| MHA -> GQA | PPL `2.017 -> 2.006`；理论 KV/token `384 -> 192`。 | 通过：本预算下缓存元素减半且 PPL 未退化，后续基线采用 GQA。 |
| coarse -> fine/shared MoE | PPL `2.071/2.081/2.039`；fine/shared 吞吐也更低。 | 分支决定：fine-grained 单独使用未通过；shared 在质量上通过但吞吐约低 35%，质量分支采用 shared，速度分支保留 coarse。 |
| aux 权重 | `arch_moe_aux` 的 PPL 最低 `2.009`；无 aux 的 load CV 为 `0.342`，aux=0.01 为 `0.075`。 | 采用 aux=0.01：质量与负载的折中最好，强 aux 没有额外质量收益。 |
| aux -> bias routing | PPL `2.009 -> 2.024`；load CV `0.075 -> 0.081`。 | 未通过：bias 在这组超参下既没有更好的主任务，也没有更均衡负载，保留 aux 并另行 sweep update rate。 |
| GQA -> 教学版 MLA | 理论 KV/token `192 -> 72`，但 PPL `2.009 -> 2.194`。 | 未通过质量门槛：保留缓存压缩假设，下一轮 sweep latent rank；不把训练显存当作解码 KV cache 证据。 |
| MTP off -> on | PPL `2.207 +/- 0.017` vs `2.196 +/- 0.034`；峰值显存 `0.197 -> 0.246 GB`。 | 仅在已被拒绝的 V3-style 分支上结论不确定；它不能决定 GQA+aux 分支是否启用 MTP。 |

分支说明：这些是组内控制变量实验，不是一条把所有胜者依次叠加的单线模型。coarse/fine/shared 与 8-routed top-2 的 aux/MLA/V3-style 机制组使用不同 MoE 拓扑；MTP 结论只适用于已被拒绝的 V3-style 分支。
